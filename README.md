# auto-create-email

Утилита для массового создания временных email-аккаунтов через консоль.

Скрипт использует доступные домены сервисов `mail.tm` и `mail.gw`, создаёт аккаунты и сохраняет результат в лог-файл в формате:

```text
email:password
```

Результаты записываются в директорию, указанную в `config.yaml`. По умолчанию это `outputs/`. Лог создаётся с уникальным именем вида `log_<uuid>.txt`.

## Возможности

- создание временных email-аккаунтов в несколько потоков;
- работа как с прокси, так и без них;
- автоматическое сохранение валидных аккаунтов в лог-файл;
- простой запуск через консольное меню.

Если `proxies.txt` пустой, приложение всё равно запускается, но работает без прокси и выводит предупреждение.

## Требования

- Python 3.12 или выше.

## Настройка `proxies.txt`

Файл с прокси указывается в `config.yaml` через параметр `proxies_list_path`. По умолчанию это `proxies.txt`.

В `proxies.txt` нужно указывать **один прокси на строку**.

Примеры:

```text
http://127.0.0.1:8080
http://login:password@127.0.0.1:8080
socks5://127.0.0.1:9050
socks5://login:password@127.0.0.1:9050
```

Без лишних пробелов, запятых и других разделителей.

## Настройка `config.yaml`

Текущая конфигурация приложения находится в `config.yaml` и содержит три основных параметра: версию приложения, путь для сохранения результатов и путь до файла с прокси.

Пример:

```yaml
app:
  version: 1.0
  output_path: outputs/
  proxies_list_path: proxies.txt
```

Что можно менять:

### `version`
Версия приложения. На работу скрипта не влияет, используется как информационный параметр.

### `output_path`
Директория, куда будут сохраняться лог-файлы с созданными почтами.

Примеры:

```yaml
output_path: outputs/
```

```yaml
output_path: logs/
```

```yaml
output_path: data/results/
```

Если папки не существует, она будет создана автоматически.

### `proxies_list_path`
Путь до файла с прокси.

Примеры:

```yaml
proxies_list_path: proxies.txt
```

```yaml
proxies_list_path: config/proxies.txt
```

```yaml
proxies_list_path: data/proxies/mobile.txt
```

Главное, чтобы файл существовал по указанному пути, иначе приложение завершится с ошибкой чтения файла.

## Запуск

После запуска приложение показывает меню:

1. Запуск
2. Выход

Далее нужно ввести количество запросов. Итоговое количество аккаунтов зависит от числа прокси и выбранного количества запросов.

В интерфейсе указано правило:

```text
Количество создаваемой почты = количество прокси * количество запросов * 2
```

Это связано с тем, что используются два сервиса, а при наличии прокси для каждого прокси создаются отдельные клиенты.

Запуск:

```bash
python main.py
```

## Формат результата

Успешно созданные аккаунты сохраняются построчно в лог-файл:

```text
example@domain.com:strongpassword123
example2@domain.com:qwerty098765
```

После завершения приложение показывает:

- количество созданных аккаунтов;
- количество неудачных попыток;
- общее время выполнения;
- среднее время на один аккаунт;
- путь до файла с результатом.

## Важные замечания

- без прокси скрипт работает, но обычно стабильность и масштаб ниже;
- файл с результатами создаётся автоматически;
- если путь в `output_path` указан корректно, нужная директория создастся автоматически;
- если путь в `proxies_list_path` неверный, приложение не сможет прочитать прокси-файл.

## Минимальная подготовка перед стартом

1. Склонируйте репозиторий.

4. Заполните `proxies.txt` прокси в формате по одному на строку.
